Understanding how Skillz connects all the pieces

üìä Key Flows Explained

üî® 1. Building & Registering Tools

WASM Tools: AI sends Rust code ‚Üí Skillz compiles to WebAssembly ‚Üí Tool registered in Tool Registry ‚Üí Available immediately

Script Tools: AI sends Python/Node.js code ‚Üí Skillz creates tool directory with dependencies ‚Üí Registers in Tool Registry ‚Üí Ready to execute

Pipelines: AI defines tool chain ‚Üí Skillz creates pipeline tool ‚Üí Automatically handles variable substitution ‚Üí Executes tools in sequence

‚ö° 2. Tool Execution Flow

Request: MCP Client calls tool ‚Üí Skillz validates & prepares context ‚Üí Runtime Engine executes

WASM: Loaded in WASI sandbox ‚Üí Direct memory access ‚Üí Fast execution ‚Üí Returns result

Script: Process spawned ‚Üí JSON-RPC over stdin/stdout ‚Üí Can call back to server ‚Üí Returns result

Response: Result formatted ‚Üí Progress/logs included ‚Üí Sent to MCP client

üîó 3. Script Tool Capabilities (Bidirectional)

Memory API: Script sends memory/set ‚Üí Server stores in libSQL ‚Üí Script sends memory/get ‚Üí Server retrieves ‚Üí TTL support

Resources API: Script sends resources/list ‚Üí Server returns available resources ‚Üí Script sends resources/read ‚Üí Server returns content

Elicitation: Script needs user input ‚Üí Sends elicitation/create ‚Üí Server forwards to MCP client ‚Üí User responds ‚Üí Script receives answer

Sampling: Script needs LLM completion ‚Üí Sends sampling/createMessage ‚Üí Server forwards to MCP client ‚Üí LLM responds ‚Üí Script receives completion

Tool Calling: Script can call other Skillz tools ‚Üí Sends tools/call ‚Üí Server executes target tool ‚Üí Returns result to caller

üî• 4. Hot Reload Flow (--hot-reload)

Watch: File watcher monitors tools directory ‚Üí Detects changes to tool files

Reload: Tool removed from registry ‚Üí Reloaded from disk ‚Üí Re-registered

Notify: MCP notification sent to clients ‚Üí tools/list_changed ‚Üí resources/updated for subscribed tools

Zero Downtime: Other tools continue working ‚Üí No server restart needed

üì¶ 5. Versioning & Rollback Flow

Auto-Backup: Tool updated ‚Üí Current version backed up to .versions/ ‚Üí Timestamp added

List Versions: Use version tool with action: "list" ‚Üí Returns all versions with timestamps

Rollback: Use version tool with action: "rollback" ‚Üí Restores selected version ‚Üí Current becomes backup

History: All versions preserved ‚Üí Can rollback multiple times ‚Üí Full version audit trail

üåê 6. HTTP/SSE Transport (--transport http)

HTTP Server: Skillz starts HTTP server ‚Üí Accepts connections from web clients

SSE Stream: Client connects to /sse ‚Üí Server sends events ‚Üí Real-time notifications

Message Endpoint: Client posts JSON-RPC to /message ‚Üí Server processes ‚Üí Response sent via SSE

Multiple Clients: Each client gets own SSE connection ‚Üí Isolated sessions ‚Üí Concurrent execution

How It Works

From code to execution in seconds

1

Write Code

AI writes Rust code for WASM tools or scripts in any language (Python, Node.js, etc.). Specify dependencies if needed.

2

Compile & Register

WASM tools are compiled using cargo. Scripts are saved with appropriate interpreter. Dependencies are installed in isolated environments.

3

Execute

WASM runs in Wasmtime sandbox. Scripts execute via subprocess with JSON-RPC 2.0 protocol using their virtual environment.

4

Persist

Tools, dependencies, and environments are stored on disk. Available across server restarts. Build your AI's toolbox over time.
